Installing Packages

BatchManager downloads package definitions from an online repository. When you run BatchManger.bat, it syncs the latest package list, locates the definition file for that package, and opens the official download page so you can grab the installer directly from its source. Metadata is then stored locally so BatchManager knows what's installed.

Creating Packages

Anyone can create a package. A package definition is a plain text file with key–value pairs describing the software and where users can download it. 

Name=Git
ID=git
Version=2.51
Author=Git Project
Description=Distributed Version Control
Homepage=https://git-scm.com
DownloadPage=https://git-scm.com/download

Save the file as <id>.ini, then add the package ID on its own line inside packages.txt so BatchManager knows the package exists. Finally, push both files to the repository.

Creating Plugins

To create a plugin, simply place a Batch file inside the plugins/ folder. BatchManager automatically detects plugins on startup — there is no installation step and no registration file to edit. 

plugins/
  cleanup.bat
  backup.bat
  systeminfo.bat

Each plugin becomes a first-class command. Delete the file to uninstall it. That's the whole system.
